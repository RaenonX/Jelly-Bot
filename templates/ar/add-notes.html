{% trans "Content Type" %}

{% blocktrans trimmed %} To let the content be valid, the content should be in the format described below corresponds to the content type. {% endblocktrans %}

{% trans "Text" %}

{% blocktrans trimmed %} Could be anything, but the length cannot be over 2000.
Markdown syntax is supported. The outcome depends on the platform. For LINE, it's support is very limited, while on Discord, it supports most basic syntax. {% endblocktrans %}

{% trans "Image" %}

{% blocktrans trimmed %} The main MIME type should be image. In other words, when you open then link, the browser should just give you an image not a web page. {% endblocktrans %}

{% trans "LINE Sticker" %}
{% url "page.doc.botcmd.cmd" code=bot_cmd_rct_msg_code as rct_msg_url %} {% blocktrans trimmed %} Should be a sticker ID. Usually the ID will be a 1~11 digits number.
The sticker ID can be acquired by:
  • Sending sticker to the bot via private channel (sending it via group channel will have no effect)
  • Calling recent activity command to lookup the sticker ID
{% endblocktrans %}
{% trans "Emoji / Sticker" %}
{% blocktrans trimmed %} Refer to the "LINE Sticker" section for LINE Sticker.
Discord emojis are treated as normal text. They are in the form of <:EMOJI_NAME:EMOJI_ID>. Example: <:neut:554302047038734366>
  • EMOJI_NAME: What you normally type in the chat window excluding the wrapping colons. If you type :neut: in the window, then "neut" is the name of the emoji.
  • EMOJI_ID: The number in the URL when you get the link of the emoji. On PC client, you can right click on the emoji and click "Copy Link" or "Open Link" to get the number / ID.
{% endblocktrans %}
